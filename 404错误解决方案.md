# 🔧 404错误彻底解决方案

## 🔍 问题诊断

你遇到的404错误是因为：
1. ❌ **后端依赖未安装**（没有node_modules文件夹）
2. ❌ **后端服务器未启动**
3. ❌ **前端请求无法到达后端**

---

## ✅ 解决步骤（必须按顺序执行）

### 第1步：打开命令提示符（CMD）

**重要：必须使用CMD，不能使用PowerShell！**

操作方法：
1. 按键盘 `Win + R`
2. 输入 `cmd`
3. 按回车键

你会看到一个黑色的窗口。

---

### 第2步：安装后端依赖

在CMD窗口中，**逐行**输入以下命令（每输入一行按一次回车）：

```cmd
cd /d e:\download\course-ai-tutor-main\course-ai-tutor-backend
```

等待命令执行完成后，继续输入：

```cmd
npm install
```

**这一步需要等待1-3分钟**，你会看到很多下载信息。

等待直到看到类似这样的信息：
```
added 150 packages in 45s
```

---

### 第3步：启动后端服务器

依赖安装完成后，继续输入：

```cmd
node src\server.js
```

如果启动成功，你会看到：
```
========================================
   🚀 Course AI Tutor - Backend Server
========================================
   环境：development
   服务地址：http://0.0.0.0:8081
   本地访问：http://localhost:8081
   API 前缀：/api
========================================
✅ 后端服务已启动
✅ 数据库已连接
✅ JWT 认证已启用
✅ 智能体联动系统已加载
========================================
```

**⚠️ 重要：保持这个CMD窗口打开，不要关闭！**

---

### 第4步：验证服务器

打开浏览器，访问：
```
http://localhost:8081/api/health
```

如果看到类似这样的JSON数据，说明服务器启动成功：
```json
{
  "status": "ok",
  "timestamp": "2026-04-19T...",
  "uptime": 123.456
}
```

---

### 第5步：刷新前端页面

回到你的前端页面，刷新浏览器（按F5），404错误应该消失了。

---

## ❓ 常见问题

### Q1: 提示"npm不是内部或外部命令"
**原因**：Node.js未安装或未添加到环境变量

**解决方法**：
1. 检查是否安装了Node.js：在CMD输入 `node -v`
2. 如果没有安装，去 https://nodejs.org 下载并安装LTS版本
3. 安装后重新打开CMD，再次尝试

---

### Q2: 提示"端口8081已被占用"
**解决方法**：
1. 打开文件 `e:\download\course-ai-tutor-main\course-ai-tutor-backend\.env`
2. 找到 `PORT=8081`，改为 `PORT=3000`（或其他端口）
3. 同时修改 `e:\download\course-ai-tutor-main\course-ai-tutor-frontend\.env` 中的端口
4. 重新启动服务器

---

### Q3: npm install很慢或失败
**解决方法**：
```cmd
npm config set registry https://registry.npmmirror.com
npm install
```

---

### Q4: 启动后还是404
**检查清单**：
1. CMD窗口是否还开着？（关闭了服务器就停了）
2. 是否看到了"后端服务已启动"的信息？
3. 浏览器访问 `http://localhost:8081/api/health` 有返回吗？
4. 前端 `.env` 文件中的端口是否是8081？

## 🎓 为什么必须用CMD？

PowerShell有执行策略限制，会阻止npm运行。CMD没有这个限制，所以必须使用CMD。
